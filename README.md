# Adversarial RL for Adaptive Eavesdropping in BB84 Quantum Key Distrubution

Code and experiment scripts for the paper _"Adversarial Reinforcement
Learning for Adaptive Eavesdropping in BB84 Quantum Key Distribution."_

This repository contains a qubit-level BB84 simulation environment and
three classical tabular RL agents (Q-Learning, SARSA, Double Q-Learning)
trained to eavesdrop while evading QBER-based detection.

**Note:** this is a classical RL project applied to a simulation of a
quantum protocol. No quantum hardware or quantum computing libraries
are required.

## Overview

BB84 quantum key distribution is secure because any eavesdropper
disturbs the quantum channel in a detectable way. Legitimate parties
monitor the error rate (QBER) and abort if it gets too high. But what
if the eavesdropper isn't naive? This project trains classical
reinforcement learning agents to play the eavesdropper, learning when
to attack and when to lie low based on QBER feedback. Even simple
tabular RL turns out to reduce detection by over 300x compared to the
best non-adaptive strategy.

Eve is modeled as an RL agent that decides, qubit by qubit, whether to
intercept or pass, based on QBER feedback announced at checkpoints.
Across 10,000 training episodes, agents learn to reduce their detection
rate far below the analytically optimal non-adaptive baseline
(Lee et al., 2022), while still extracting information.

For full results, statistical analysis, and discussion, see the paper
(`paper/main.pdf` or the arXiv link above).

## Repository Structure

```
.
├── bb84_simulator.py                 # Core qubit-level BB84 simulation
├── bb84_environment.py               # Gym-style RL environment wrapper
├── agents.py                         # Q-Learning, SARSA, Double Q-Learning implementations
├── training.py                       # Single-run training script
├── baselines.py                      # Always Attack and Fixed Rate baselines
├── extension_random_checkpoints.py   # Randomized checkpoint extension experiment
├── multiseed.py                      # Multi-seed (5 seeds) experiment runner
├── plots.py                          # Figure generation
├── results/                          # Output CSVs and figures (generated by scripts)
└── paper/                            # LaTeX source for the paper
```

pip install -r requirements.txt

## Usage

### Run a single training run

```bash
python training.py --agent q_learning --noise 0.01 --seed 42
```

### Run the full multi-seed experiment (45 runs, ~50 min)

```bash
python multiseed.py
```

### Run the random-checkpoint extension

```bash
python extension_random_checkpoints.py
```

### Generate figures

```bash
python plots.py
```

Output figures are saved to `results/figures/`.

## Reproducing Paper Results

The exact configuration used in the paper:

- 3 agents × 3 noise levels (μ_ch ∈ {1%, 3%, 5%}) × 5 seeds (42, 123, 456, 789, 1337)
- 10,000 training episodes per run
- Hyperparameters as listed in Table 1 of the paper

Running `multiseed.py` followed by `plots.py` reproduces all figures
and tables in the Results section.
